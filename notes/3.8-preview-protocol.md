# 3.8 Browser Preview — URL checklist (from asar templates)

Source: `builtin/preview/static/views/script.ejs`, `preview-app/src/main.ts`, `builtin/server/package.json`

## Boot scripts (page)

| URL / placeholder | Purpose |
|-------------------|---------|
| `/scripting/polyfills/bundle.js` | polyfills |
| `/scripting/engine/bin/.cache/dev/preview/import-map.json` | engine feature import-map |
| `<%=packImportMapURL%>` | project modules import-map |
| `/scripting/import-map-global` | map `cc`, `cc/env` |
| `/scripting/systemjs/system.js` | SystemJS |
| `<%=packResolutionDetailMapURL%>` | resolution detail map (fetched) |
| `/scripting/engine/bin/.cache/dev/preview/bundled/index.js` | engine bundle for preview |
| `<%=settingsJs%>` | loads `window._CCSettings` |
| `/preview-app/index.js` | preview bootstrap UI + `cc.game` launch |

## Runtime fetches (preview-app)

| URL | Purpose |
|-----|---------|
| `scene/${launchScene}.json` | launch scene JSON |
| `/missing-asset/${uuid}` | missing asset diagnostics |

## Transport

- HTTP preview server default port: **7456**
- `socket.io` (`window.socket=io()` in older `init.js`; preview error channel)

## Compare with Cocos4 / cocos-cli

cocos-cli exposes similar surfaces:

- `/preview/settings.js` -> `_CCSettings`
- `/scripting/*` engine + pack chunks
- live-reload on `programming` compiled

3.8 uses engine cache path under `/scripting/engine/bin/.cache/dev/preview/...` which is editor-generated; headless must reproduce or reuse that cache layout.
